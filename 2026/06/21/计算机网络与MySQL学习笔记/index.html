计算机网络

1. HTTP 和 HTTPS

  • HTTP（Hypertext Transfer Protocol）：超文本传输协议
  • HTTPS（Hypertext Transfer Protocol Secure）：超文本传输安全协议

1.1 HTTP

HTTP（Hypertext Transfer Protocol），是一种发送和接收 HTML 页面的方法，主要用于 Web 浏览器和网站服务器之间传递信息。它的主要特点如下：

  1. 基于请求响应模式：HTTP 协议采用客户端-服务器架构模式，客户端向服务器发送请求，服务器返回相应的响应。这种模式能有效分离应用逻辑，提高系统的可维护性和扩展性。
  2. 基于文本传输：HTTP 协议使用 ASCII 码作为通信协议，每个请求和响应都是一条文本消息,这使得通信协议更加简单、直观、易于处理。
  3. 支持多媒体传输：HTTP 协议可以传输多种类型的数据，如 HTML、XML、JSON、图片、音频、视频等，这使得 HTTP 协议成为一种通用的网络传输协议。
  4. 无连接：HTTP 协议是一个无连接协议，每个请求都是独立的，服务器处理请求后立即关闭连接。这有助于节省资源，但也带来了一些缺点，如需要重新建立连接、重复发送相同的头部信息等。
  5. 无状态：HTTP 协议没有客户端的状态存储，也没有事务处理的”内存”能力。这意味着每次访问网站时可能需要重复的登录操作。

1.2 HTTPS

HTTPS（Hypertext Transfer Protocol Secure） 在 HTTP 的基础上通过传输加密和身份认证，保证了传输过程的安全性。HTTPS 的内容加密、身份验证以及数据完整性保护的原理主要依赖于 SSL/TLS 协议。

HTTPS 使用对称加密和非对称加密相结合的方式来实现内容加密。

对称加密：在密钥交换完成后，客户端和服务器会生成一个共享的会话密钥。这个会话密钥用于后续的加密和解密操作。双方使用这个会话密钥，通过对称加密算法（如 AES）对传输的数据进行加密和解密。

非对称加密：在 HTTPS 的握手阶段，服务器会将其公钥发送给客户端。客户端使用服务器的公钥加密一个随机数，然后将加密后的随机数发送给服务器。服务器使用其私钥解密这个随机数，从而确保双方都能安全地交换密钥。

HTTPS 使用数字证书来验证服务器的身份。

数字证书由权威的证书颁发机构（CA）颁发，包含服务器的公钥、服务器的身份信息以及 CA 的签名等。当客户端与服务器建立 HTTPS 连接时，服务器会将其数字证书发送给客户端。

验证过程：

  1. 客户端会检查证书是否由受信任的 CA 颁发
  2. 检查证书是否过期，以及证书中的服务器身份信息是否与正在连接的服务器一致
  3. 使用 CA 的公钥验证证书上的签名，确保证书在传输过程中没有被篡改

如果证书验证通过，客户端就可以确认服务器的身份是可信的。

一句话总结：HTTPS = HTTP + TLS。非对称加密用于安全地交换会话密钥，对称加密用于实际的数据传输（兼顾安全和性能）。

MySQL

MySQL 是一种开放源代码的关系型数据库管理系统（RDBMS），使用最常用的数据库管理语言——结构化查询语言（SQL）进行数据库管理。

1. MySQL 常用的存储引擎有什么？有什么区别？

  • InnoDB：MySQL 的默认存储引擎，支持事务、行锁和外键等操作。
  • MyISAM：MySQL 5.1 版本前的默认存储引擎，并发性比较差，不支持事务和外键，默认锁粒度为表级锁。

2. MySQL 数据类型

2.1 整型

整数类型一共有 5 种：TINYINT、SMALLINT、MEDIUMINT、INT（INTEGER）和 BIGINT。

整数类型 字节 有符号取值范围 无符号取值范围
TINYINT 1 -128 ~ 127 0 ~ 255
SMALLINT 2 -32768 ~ 32767 0 ~ 65535
MEDIUMINT 3 -8388608 ~ 8388607 0 ~ 16777215
INT、INTEGER 4 -2147483648 ~ 2147483647 0 ~ 4294967295
BIGINT 8 -9223372036854775808 ~ 9223372036854775807 0 ~ 18446744073709551615

可选属性：

  • M：显示宽度，取值范围 (0, 255)。例如 int(5) 表示数据宽度小于 5 位时在数字前面填满宽度，需要配合 ZEROFILL 使用。

    常见疑问：如果设置了显示宽度，插入数据宽度超过限制会被截断或失败吗？

    答：不会。显示宽度与类型可存储的值范围无关，仍然按类型实际宽度保存。

  • UNSIGNED：无符号类型。int 默认显示宽度为 int(11)，无符号 int 默认为 int(10)。

  • ZEROFILL：0 填充。不够 M 位时左侧补 0；超过 M 位时只要不超过数据存储范围即可。

2.2 浮点型

FLOAT（4 字节）和 DOUBLE（8 字节）。

精度说明：

  • FLOAT(M, D) 或 DOUBLE(M, D)：M 为精度，D 为标度，M = 整数位 + 小数位，D = 小数位。要求 D <= M <= 255, 0 <= D <= 30。
  • 不管是否显式设置精度，整数部分超出范围会报错；小数部分四舍五入后超出范围才报错，否则只给警告。

重要原则：浮点数是不准确的，避免使用 = 判断两个浮点数是否相等。

2.3 定点数

数据类型 字节数 含义
DECIMAL(M, D) M + 2 字节 有效范围由 M 和 D 决定

DECIMAL 是精确的，底层使用字符串形式储存。

2.4 字符串类型

主要有 CHAR 和 VARCHAR：

字符串类型 特点 长度范围 占用的存储空间
CHAR(M) 固定长度 0 <= M <= 255 M 个字节
VARCHAR(M) 可变长度 0 <= M <= 65535 (实际长度 + 1) 个字节

什么时候用 CHAR 比 VARCHAR 好？

  • 存储很短的信息（VARCHAR 还要占 1 字节存长度）
  • 固定长度的字段，比如 UUID 作为主键
  • 十分频繁修改的列（VARCHAR 每次存储要额外计算长度）

2.5 日期与时间类型

类型 名称 字节 格式 最小值 最大值
YEAR 年 1 YYYY 或 YY 1901 2155
TIME 时间 3 HH:MM:SS -838:59:59 838:59:59
DATE 日期 3 YYYY-MM-DD 1000-01-01 9999-12-03
DATETIME 日期时间 8 YYYY-MM-DD HH:MM:SS 1000-01-01 00:00:00 9999-12-31 23:59:59
TIMESTAMP 日期时间 4 YYYY-MM-DD HH:MM:SS 1970-01-01 00:00:00 UTC 2038-01-19 03:14:07 UTC
点击展开：其他数据类型（ENUM / SET / JSON / 空间）
  • BIT(M)：位类型，长度 1 <= M <= 64，占用约 (M + 7) / 8 字节
  • ENUM：枚举类型，最多 65535 个成员
  • SET：集合类型，最多 64 个成员
  • 二进制字符串类型：BINARY、VARBINARY、TINYBLOB、BLOB、MEDIUMBLOB、LONGBLOB
  • JSON 类型：5.7+ 原生支持
  • 空间类型：GEOMETRY、POINT、LINESTRING、POLYGON 等

3. 数据存储和索引

索引是对数据库表的一列或多列的值进行排序的一种结构，使用索引可以快速访问数据表中的特定信息。

3.2 索引的数据结构

索引的数据结构主要有 B+ 树和哈希表。InnoDB 的默认索引类型为 B+ 树索引。

为什么 InnoDB 选择 B+ 树而不是 B 树或哈希表？

  • 相比 B 树：B+ 树非叶子节点不存数据，每个节点能容纳更多索引项，树高更低，I/O 次数更少；叶子节点用链表串联，范围查询效率高。
  • 相比哈希表：哈希表只能精确匹配，不支持范围查询和排序。
